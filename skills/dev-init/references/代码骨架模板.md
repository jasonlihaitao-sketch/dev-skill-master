> 通用骨架，初始化时按实际技术栈剪裁后拷到仓库根 `src/`。
> 技术栈不预设，下面给出最小可跑的结构占位与示例模块，dev-implement 往里填。

## 目录结构（示例，按技术栈调整）

```text
src/
├── index.{ext}            # 入口
├── config/                # 配置加载
├── modules/               # 业务模块，一个 feature 一个子目录
│   └── example/
│       ├── example.{ext}        # 实现
│       └── example.test.{ext}   # 测试（TDD 先写这里）
└── shared/                # 跨模块共享：类型、工具、错误
```

## 约定

- 一个 feature 对应 `src/modules/{feature}/`，与 `develop/features/{feature}/` 对应
- 测试与实现同目录或 `tests/` 镜像，遵循 project-context.md 第 5 章规范
- 入口、配置、共享代码分层，业务模块互不反向依赖
