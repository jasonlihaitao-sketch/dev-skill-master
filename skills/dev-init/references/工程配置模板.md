# 工程配置模板（通用）

> 初始化时落地，保证 dev-implement 的 TDD 与 dev-review 有自动化质量基线。
> 按技术栈选对应工具，下面给出通用清单与配置占位。

## 必备项清单

- [ ] **lint**：代码规范检查（对应 project-context.md 第 5 章）
- [ ] **format**：统一格式化
- [ ] **test runner**：TDD 红/绿判定依赖
- [ ] **git hooks**（pre-commit）：提交前跑 lint + 测试
- [ ] **CI**：push/PR 跑 lint + 测试 + build
- [ ] **.gitignore**：忽略构建产物、依赖、本地环境

## 配置占位

### lint / format

```text
# {lint 配置文件}
# 待办：按 project-context.md 第 5 章命名/格式规范配置
```

### 测试运行

```text
# 测试命令：{test-runner} {pattern}
# 约定：单测与实现同目录或 tests/ 镜像
```

### pre-commit hook

```text
# .git/hooks/pre-commit 或 husky/lefthook 配置
# 提交前执行：lint + 受影响测试
```

### CI

```text
# {CI 配置文件}
# 作业：install → lint → test → build
# 失败即阻断合并
```

### .gitignore（通用最小集）

```text
# 依赖
node_modules/
.venv/
vendor/
# 构建产物
dist/
build/
target/
# 本地环境
.env
.env.local
# IDE
.idea/
.vscode/
```
