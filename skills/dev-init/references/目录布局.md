# 目录布局

`develop/` 只放流程产物，实际代码留仓库根（遵循语言约定）。

```text
repo/
├── src/                                    # 实际代码（语言约定，初始化时从代码骨架拷出）
├── tests/                                  # 测试代码
├── .lint/ .ci/ 等                          # 工程配置（初始化时落地）
│
└── develop/                                # 流程工作区，只放流程产物
    ├── project-context.md                  # 全局上下文（无 feature 前缀）
    ├── adr/                                # 全局架构决策（跨功能）
    │   └── adr-01-{slug}.md
    └── features/
        └── {feature}/                      # 一个 feature 一个目录
            ├── {feature}-requirement.md
            ├── {feature}-design.md
            ├── adr/                        # 功能级决策
            │   └── {feature}-adr-01-{slug}.md
            ├── tasks/
            │   └── {feature}-task-01-{slug}.md
            └── {feature}-review.md
```

## 命名规则

| 文件 | 命名 | 归属 |
|---|---|---|
| 全局上下文 | `project-context.md` | develop/ |
| 全局 ADR | `adr-01-{slug}.md` | develop/adr/ |
| 需求 | `{feature}-requirement.md` | develop/features/{f}/ |
| 方案 | `{feature}-design.md` | develop/features/{f}/ |
| 功能级 ADR | `{feature}-adr-01-{slug}.md` | develop/features/{f}/adr/ |
| 任务 | `{feature}-task-01-{slug}.md` | develop/features/{f}/tasks/ |
| review 报告 | `{feature}-review.md` | develop/features/{f}/ |

- 全部小写
- feature 级文件带 `{feature}-` 前缀
- 编号两位起（`01`、`02`…）
- slug 用 kebab-case，有意义（如 `token-strategy`，非 `t1`）

## 各 skill 读写映射

| skill | 读取 | 产出 |
|---|---|---|
| dev-init | 仓库根现状 | develop/ 骨架、project-context.md、代码骨架、工程配置、CLAUDE.md 引用 |
| dev-requirement | project-context.md | features/{f}/{f}-requirement.md |
| dev-design | {f}-requirement.md、project-context.md | {f}-design.md、adr/ |
| dev-implement | {f}-design.md、{f}-requirement.md、project-context.md | tasks/{f}-task-NN.md、src/、tests/ |
| dev-review | project-context.md、{f}-design.md、{f}-requirement.md、tasks/、src/、tests/ | {f}-review.md |
