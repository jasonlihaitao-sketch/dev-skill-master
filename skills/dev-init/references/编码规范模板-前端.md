# 编码规范模板 — 前端（以 React / Vue / TypeScript 为主）

> 适用前提：本模板是「前端 React/Vue + TypeScript 栈」的规范参考。**仅当项目第 2 章技术栈判定为该前端栈时取用**；
> 若项目前端是其它栈（如原生 JS、非 TS、其它框架）且无对应模板，则**不套用本模板、不强制**，按项目自身约定执行。
> 跨端通用项（commit 规范、lint/format 工具位置）见 `project-context.md` 第 5 章，不在本文件重复。
>
> 用法：逐字段填充，无内容写「待定」并列入待澄清；范例仅供示范，按项目实际裁剪。React/Vue 二选一时，按项目实际框架取对应条目。

## 1. 命名

> 文件 / 组件 / 变量 / 常量 / 类型 / CSS 类命名约定。

- **文件名**：（组件文件大驼峰 `UserCard.tsx`；非组件小驼峰或 kebab-case）
- **组件名**：（大驼峰，名词，如 `UserList`）
- **变量/函数**：（小驼峰，`handleSubmit`）
- **常量**：（全大写下划线 `MAX_COUNT`）
- **类型/接口**：（大驼峰，接口可加或不加 `I` 前缀，统一即可）
- 范例：React 组件文件 `UserCard.tsx` 导出 `export function UserCard()`；Vue SFC 文件 `UserCard.vue`。

## 2. 组件结构

> 组件拆分粒度与组织。

- **React**：（函数组件 + Hooks；单文件单组件；props 用类型/接口定义；副作用集中在自定义 Hook）
- **Vue**：（SFC `<script setup>`；props 用 `defineProps<T>`；组合式函数 `useXxx`）
- **职责**：（容器组件 vs 展示组件分离；组件行数上限，如 ≤ 200 行）
- 范例：React 容器组件拉数据、展示组件纯渲染；复杂表单逻辑抽 `useUserForm()` Hook。

## 3. 状态管理

> 本地状态 / 全局状态 / 服务端状态。

- **本地状态**：（`useState` / `ref`；能局部就不上全局）
- **全局状态**：（Zustand / Redux / Pinia，按项目实际；说明何时用全局）
- **服务端状态**：（React Query / SWR / TanStack Query，请求缓存与失效策略）
- 范例：用户登录态用全局 store；列表数据用 React Query，按 queryKey 失效；表单临时态用本地 `useState`。

## 4. 类型（TypeScript）

> 类型安全约定。

- **严格模式**：（`strict: true`；禁止 `any`，必须用 `unknown` + 收窄或显式类型）
- **类型来源**：（API 响应类型从后端 schema 生成或手写，是否用 OpenAPI codegen）
- **共享类型**：（跨组件类型放 `types/`，避免重复定义）
- 范例：`type User = { id: string; name: string }`；接口返回 `Promise<User>` 而非 `Promise<any>`。

## 5. 样式

> 样式方案与约定。

- **方案**：（Tailwind / CSS Modules / styled-components / Vue `<style scoped>`，按项目实际）
- **约定**：（避免内联样式堆积；设计 token 复用；响应式断点统一）
- 范例：Tailwind 优先用 token 类；主题色走 CSS 变量不硬编码 `#fff`。

## 6. 数据获取与接口

> 前后端交互约定。

- **请求层**：（统一封装 fetch/axios 客户端；baseURL / 拦截器 / 错误处理集中）
- **错误处理**：（网络错误 / 业务错误码统一处理；loading/error 状态显式管理）
- **环境变量**：（`import.meta.env` / `process.env`，敏感配置不入前端代码）
- 范例：`apiClient.get<User>('/users/1')`，401 由拦截器统一跳登录；接口类型与后端契约对齐。

## 7. 可访问性（a11y）

> 语义化与无障碍。

- **语义化**：（用 `<button>` 不用 `<div onClick>`；表单配 `label`；图片 `alt`）
- **键盘**：（可交互元素可 Tab 聚焦；模态焦点陷阱）
- 范例：图标按钮带 `aria-label`；表单输入绑 `htmlFor`/`id`。

## 8. 测试约定

> 与 dev-implement TDD 对齐（前端 UI 视图层可酌情，须注明替代验证）。

- **单元测试**：（纯函数 / Hook / 工具函数，Vitest / Jest，必测）
- **组件测试**：（React Testing Library / Vue Test Utils，测行为不测实现）
- **E2E**：（Playwright / Cypress，覆盖关键用户流）
- **UI 视图层**：（难测的可酌情，注明「前端 UI，TDD 酌情」并给替代验证，如手动走查清单）
- 范例：`UserCard.test.tsx` 测「传入 user 渲染名字」；表单交互用 RTL 测提交回调。

## 9. 性能

> 渲染与加载性能。

- **渲染**：（React `useMemo`/`useCallback`/`React.memo` 按需；Vue `computed`/`shallowRef` 按需；避免无意义重渲染）
- **加载**：（路由懒加载；大依赖动态 import；图片懒加载）
- 范例：路由级 `React.lazy` + `Suspense`；列表项 `memo` 防重排。

## 10. 安全

> 与 dev-review 安全维度对齐。

- **XSS**：（禁止 `dangerouslySetInnerHTML`/`v-html` 直渲用户输入；输出转义）
- **敏感信息**：（token 存储位置与过期；不把密钥放前端）
- **依赖**：（定期审计 `npm audit`；锁定版本 `package-lock`）
- 范例：富文本用白名单清洗后再渲染；token 用 httpOnly cookie 或短时内存存储。
